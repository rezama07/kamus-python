BAB 17: JALUR PERAKITAN OTOMATIS

Bab 17 / Kamus Python (Version Control & CI/CD)

Bayangkan dapur restoran Anda sekarang memiliki 5 orang koki yang sedang membuat satu resep rahasia bersama-sama. Bagaimana caranya agar jika satu koki melakukan kesalahan (misalnya menuangkan terlalu banyak garam), resep tersebut tidak rusak selamanya? Bab ini adalah kunci kolaborasi tingkat industri.

📖 KATA 103: Git & GitHub (dibaca: git dan git-hab / artinya: Mesin Waktu & Gudang Kode)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Git: Mesin waktu yang dipasang di dalam laptop Anda untuk merekam setiap perubahan kode yang Anda ketik (detik demi detik), sehingga Anda bisa membatalkan kesalahan kapan saja.
GitHub: Gudang raksasa di internet tempat Anda dan tim menyimpan "buku resep" (kode) tersebut agar bisa dikerjakan bersama dari berbagai laptop yang berbeda.

💡 ANALOGI DUNIA NYATA:

Bayangkan Anda sedang menulis buku resep di Microsoft Word. Daripada Anda menyimpan file dengan nama Resep_Final.docx, lalu Resep_Final_Revisi.docx, lalu Resep_Final_Revisi_Banget.docx, Git akan secara otomatis menyimpan seluruh sejarah riwayat tulisan Anda dalam satu file rapi. GitHub adalah Google Drive-nya, tempat file itu ditaruh agar bisa diedit bersama-sama.

⚠️ ATURAN WAJIB:

Sebelum mengunggah kode rahasia perusahaan ke GitHub, pastikan Anda tidak ikut mengunggah kunci brankas (seperti API Key di Kata 66). Gunakan fitur pelindung yang disebut .gitignore.

💻 CONTOH KODE (Perintah Terminal):

# Merekam perubahan kode di laptop (Save)
git commit -m "menambahkan fitur balas chat AI"

# Mengunggah kode ke Gudang Internet (GitHub)
git push origin main

📖 KATA 104: Pull Request (PR) (dibaca: pul ri-kues / artinya: Minta Izin Penggabungan)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Proses mengajukan izin secara resmi kepada "Ketua Koki" (Lead Engineer) agar kode baru yang baru saja Anda ketik boleh digabungkan ke dalam buku resep utama milik perusahaan.

💡 ANALOGI DUNIA NYATA:

Anda adalah koki magang yang baru saja menciptakan saus sambal baru. Anda tidak boleh langsung mencampurkan saus itu ke dalam panci utama yang akan disajikan ke pelanggan. Anda harus menaruhnya di mangkuk kecil, lalu bilang ke Kepala Koki: "Chef, ini saus buatanku (Pull Request), tolong dicicipi. Kalau enak, baru boleh dicampur ke panci utama ya."

⚠️ ATURAN WAJIB:

Di perusahaan teknologi yang bagus, seorang Engineer pemula (Junior) diharamkan menekan tombol "Merge" (Gabung) pada PR-nya sendiri. Harus Engineer lain yang me-review (mencicipi) kodenya.

💻 CONTOH PEMAKAIAN:

Tidak ada kode Python. PR dilakukan dengan cara meng-klik tombol "Create Pull Request" di dalam website GitHub.

📖 KATA 105: CI/CD (Continuous Integration / Continuous Deployment) (dibaca: si-ai si-di / artinya: Ban Berjalan Otomatis)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Robot pabrik (seperti GitHub Actions) yang akan langsung bekerja secara otomatis setiap kali Anda selesai melakukan Pull Request atau mengunggah kode baru. Robot ini akan melakukan testing (menguji apakah kodenya error), membungkusnya ke Docker (Kata 83), dan langsung melemparkannya ke server AWS (Kata 88) tanpa Anda perlu menekan tombol apa-apa lagi.

💡 ANALOGI DUNIA NYATA:

Begitu Kepala Koki mencicipi dan menyetujui saus buatan Anda (PR disetujui), sebuah mesin ban berjalan (Conveyor Belt) akan otomatis menyala. Mesin itu akan menyedot saus Anda, membotolkannya, menempelkan label, dan langsung mengirimkannya ke ribuan minimarket di seluruh negeri detik itu juga. Anda tinggal duduk santai minum kopi.

⚠️ ATURAN WAJIB:

Jangan pernah mengaktifkan robot "Deploy Otomatis" (CD) jika Anda belum membuat robot "Testing Otomatis" (CI) yang kuat. Kalau kodenya error dan langsung ter-deploy otomatis, seluruh website perusahaan bisa mati seketika.

💻 CONTOH KODE:

Potongan skrip instruksi untuk robot GitHub Actions yang disimpan dalam file berakhiran .yml:

# Contoh instruksi ke robot
steps:
  - name: Tes kodingan AI (Continuous Integration)
    run: python test_bot.py
  - name: Lempar ke Server (Continuous Deployment)
    run: kirim_ke_aws_sekarang()
