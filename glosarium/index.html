GLOSARIUM PYTHON

Kumpulan Lengkap Istilah & Bahasa Gaul Pemrograman

Pemula sering bingung bukan karena kodenya, tapi karena istilah gaul/slang yang diucapkan oleh para senior atau mentor di kelas. Bagian ini khusus dibuat untuk menerjemahkan bahasa gaul tersebut. Saat Anda masuk ke komunitas coding, Anda pasti akan sering mendengar kata-kata ini:

1. Ngoding / Coding

Kegiatan mengetik instruksi (resep) untuk komputer.

2. Run / Execute

Tombol sakelar untuk menyuruh komputer "Ayo jalan! Kerjakan tugas yang baru saja saya ketik!".

3. Bug (dibaca: Bag)

Harafiahnya berarti Serangga/Kutu. Di dunia IT, ini artinya ada "kecacatan" atau kesalahan logika di dalam kode Anda yang membuat program berjalan tidak semestinya.

4. Debugging (dibaca: Di-ba-ging)

Kegiatan mencari dan membasmi si "Kutu" (Bug) tadi sampai programnya lancar kembali. Kegiatan ini memakan 80% waktu seorang programmer.

5. Deploy (dibaca: Di-ploy)

Meng-online-kan aplikasi. Membawa program Anda dari laptop pribadi Anda, ke internet agar bisa dipakai oleh seluruh orang di dunia.

6. Library / Package

"Kotak peralatan ekstra" buatan orang lain yang bisa Anda pinjam ke dalam kode Anda (lewat perintah import), agar Anda tidak perlu membuat semuanya dari nol.

7. API (Application Prog. Interface)

Jembatan komunikasi atau "Pelayan Pembawa Pesan". Misalnya, kode di laptop Anda menyuruh API pergi ke server Google untuk meminta jawaban AI, lalu API membawa jawaban itu kembali ke layar Anda.

8. Terminal / CLI

Layar hitam berisi teks putih tempat Anda mengetikkan perintah langsung ke "jantung" sistem komputer.

9. Repo / Repository

Gudang penyimpanan digital (biasanya menggunakan website GitHub) untuk menyimpan seluruh file kodingan Anda agar aman dan bisa dikerjakan bersama tim.

10. Local vs Production

Lokal berarti program Anda baru bisa berjalan aman di laptop sendiri. Production berarti program tersebut sudah rilis ke internet dan dipakai oleh pengguna sungguhan.

11. Prompting

Seni menyusun kalimat perintah (prompt) yang spesifik kepada AI agar AI memberikan jawaban yang akurat dan tidak berhalusinasi.

12. Frontend (Front-end)

Bagian "Etalase". Semua tampilan seperti tombol, warna, dan teks di aplikasi yang bisa dilihat langsung oleh pengguna.

13. Backend (Back-end)

Bagian "Dapur". Mesin tak terlihat di belakang layar yang sibuk menghitung data dan berpikir.

14. Server

Komputer super yang menyala 24 jam tanpa henti untuk menaruh aplikasi kita agar selalu bisa diakses orang lain lewat internet.

15. Database (DB)

Gudang digital raksasa tempat menyimpan semua data (seperti nama user, riwayat chat, atau dokumen) secara permanen.

16. Open Source

Program gratisan yang "resep" kodingannya dibuka untuk umum. Siapa saja boleh melihat, mencontek, atau memodifikasinya tanpa perlu membayar lisensi.

17. Token

Hitungan "potongan kata" yang dibaca oleh AI. Semakin panjang Anda mengobrol dengan AI, semakin banyak token yang dikuras.

18. Environment (Env)

Ruangan kerja terisolasi di dalam laptop. Tujuannya agar alat-alat masak (Library) untuk Proyek A tidak tercampur dan merusak alat-alat di Proyek B.

19. Framework

Kerangka kerja setengah jadi. Ibarat membangun rumah, pondasi dan tiangnya sudah dibikinkan, Anda tinggal mengisi temboknya saja agar lebih cepat selesai.

20. Model (Model AI)

"Otak" digital dari sebuah AI yang sudah selesai diajari untuk melakukan tugas tertentu (seperti menjawab pertanyaan, menerjemahkan bahasa, atau mengenali gambar).

21. LLM (Large Language Model)

Jenis Model AI raksasa yang dikhususkan hanya untuk memahami dan memproduksi teks bahasa manusia (contohnya ChatGPT, Gemini, atau Claude).

22. Dataset

Kumpulan data mentah (bisa berupa jutaan teks, tabel, atau dokumen) yang digunakan sebagai "buku pelajaran" untuk memberi makan dan melatih AI.

23. RAG (Retrieval-Augmented Gen.)

Teknik memberi "buku contekan" kepada AI. RAG memaksa AI membaca dokumen rahasia kita dulu sebelum menjawab agar tidak halusinasi.

24. Cloud (Komputasi Awan)

Menyewa komputer super canggih milik perusahaan besar lewat internet, sehingga kita tidak perlu membeli komputer/server fisik yang mahal.

25. Git & GitHub

Git adalah alat "mesin waktu" untuk merekam perubahan kode. GitHub adalah semacam Google Drive khusus programmer untuk menyimpan dan memamerkan kode ke internet.

26. Algoritma (Algorithm)

Urutan langkah-langkah logis untuk menyelesaikan suatu masalah. Ibarat resep masakan dari tahap persiapan hingga makanan siap dihidangkan.

27. IDE / Code Editor

Perangkat lunak atau aplikasi tempat Anda mengetik, mengedit, dan menjalankan kode (contoh: Visual Studio Code, PyCharm).

28. PIP (Python Package Installer)

"Toko aplikasi" bawaan Python. Alat ini bertugas untuk mengunduh dan memasang Library tambahan dari internet ke dalam laptop Anda.

29. Hardcode (Hard-coding)

Praktik menuliskan data secara permanen langsung ke dalam kode. (Contoh: Menulis nama "Budi" langsung di kode, ketimbang memakai input()).

30. Clean Code

Praktik menulis kodingan yang rapi, terstruktur, dan mudah dipahami oleh manusia (programmer lain), bukan hanya sekadar bisa dibaca oleh komputer.

31. Script / Scripting

Sebuah file tunggal (biasanya berakhiran .py) yang berisi rentetan perintah kode Python untuk melakukan satu tugas otomatis.

32. Web Scraping

Teknik membuat program Python yang bisa menyedot, menyalin, dan mengumpulkan data dari suatu halaman website secara otomatis.

33. OOP (Object-Oriented Prog.)

Gaya penulisan kode yang menganggap komponen di dalam program sebagai "Benda/Objek" dunia nyata (misal: objek "Mobil" punya warna dan punya mesin).

34. Path / Directory

Alamat pasti lokasi sebuah file atau folder di dalam sistem komputer Anda (contoh: C:/Dokumen/proyek_python/).

35. Syntax Sugar

Cara penulisan kode di Python yang dirancang sangat singkat dan manis (mudah dibaca), untuk menggantikan cara penulisan lama yang panjang dan rumit.

36. Virtual Environment

Ruangan kerja terisolasi di dalam laptop (seperti venv atau conda). Tujuannya agar alat-alat masak (Library) antar Proyek tidak saling merusak.

37. PEP 8 (dibaca: pep-eight)

Buku pedoman resmi tata krama penulisan Python sedunia. Jika kode tidak PEP 8, artinya kodingan Anda berantakan dan tidak sesuai standar internasional.

38. JSON (JavaScript Object Not.)

Format teks universal untuk tukar-menukar data di internet. Bentuknya 99% mirip dengan Dictionary di Python (memakai { }).

39. Refactoring (Refaktor)

Kegiatan "bersih-bersih bengkel". Merapikan dan menyederhanakan kode yang sudah jalan agar lebih cepat dan enak dibaca, tanpa mengubah hasil akhirnya.

40. Try-Except (Error Handling)

Sabuk pengaman dalam coding. "Coba (Try) jalankan ini, kalau gagal/error, Tangkap (Except) dan jalankan rencana cadangan" agar program tidak mati total.

41. Boilerplate

Teks "Kop Surat" atau template kode standar yang panjang dan terpaksa harus diketik berulang-ulang setiap kali membuat proyek baru.
