🚦 HALAMAN NOL: Rambu Lalu Lintas Python

Di jalan raya, Anda melihat lampu merah berarti berhenti. Di Python, komputer juga punya rambu lalu lintas berupa tanda baca. Jika Anda salah menaruh tanda, komputer akan kecelakaan (Error). Berikut adalah 5 rambu paling sering muncul:

1. Tanda Sama Dengan ( = ) → "TUKANG ISI"

Di matematika, = artinya hasil akhir. Di Python, = artinya "memasukkan data ke dalam kotak".

nama = "Budi"
(Artinya: Masukkan teks Budi ke dalam kotak bernama nama)

2. Tanda Koma ( , ) → "PEMISAH ANTREAN"

Komputer tidak bisa membedakan mana barang pertama dan kedua jika digabung. Koma wajib dipakai untuk memisahkan data yang berjejer agar tidak saling tabrak.

["Apel, "Jeruk", "Mangga"]

3. Tanda Kutip ( " " atau ' ' ) → "PENANDA TEKS"

Rambu wajib untuk memberi tahu komputer, "Hei, ini cuma teks biasa ya, tolong jangan dihitung secara matematika!" (Ini disebut String).

4. Tanda Pagar ( # ) → "TUTUP MATA"

Rambu khusus untuk komputer agar pura-pura tidak melihat tulisan di sebelahnya. Gunakan ini untuk menulis pesan rahasia ke sesama manusia. (Ini disebut Comment).

5. Tanda Kurung ( (), [], {} ) → "BUNGKUSAN BEDA FUNGSI"

Python itu cerewet soal bungkusan.

  • [ ] (Kurung Siku) untuk daftar biasa (List).
  • ( ) (Kurung Biasa) untuk daftar yang dikunci mati (Tuple).
  • { } (Kurung Kurawal) untuk daftar yang berpasangan (Dictionary).

BAB 1: BAHAN BAKU & WADAH

Bab 1 / Kamus Python (Data & Variabel)

📖 KATA 1: Variable (dibaca: Variabel)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Tempat (kotak) untuk menyimpan data agar bisa dipanggil lagi nanti.

💡 ANALOGI DUNIA NYATA:

Bayangkan sebuah kotak kardus kosong. Anda memasukkan sepasang sepatu ke dalamnya, lalu di luar kotaknya Anda tempelkan stiker nama bertuliskan "Sepatu_Olahraga". Stiker nama itu adalah Variable, dan sepatu di dalamnya adalah datanya.

⚠️ ATURAN WAJIB:

Di dalam kodingan, Anda TIDAK PERLU menulis kata "Variable". Anda cukup langsung membuat nama stikernya (nama kotaknya). Nama tidak boleh memakai spasi (gunakan garis bawah _).

💻 CONTOH KODE:

nama_peserta = "Budi"

# (Keterangan: Di sini, kata nama_peserta itulah yang disebut sebagai Variable. Sedangkan "Budi" adalah isi di dalam kotaknya).

📖 KATA 2: String (dibaca: string)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Teks biasa. Apapun yang berupa teks, huruf, kata, atau kalimat di dalam Python disebut String.

💡 ANALOGI DUNIA NYATA:

Bayangkan String itu seperti tulisan di atas secarik kertas label. Komputer tidak akan menghitung tulisan itu secara matematika, komputer hanya membacanya persis seperti apa yang tertulis.

⚠️ ATURAN WAJIB:

Untuk memberi tahu Python bahwa sesuatu itu adalah String (teks), Anda wajib mengapit teks tersebut dengan tanda kutip dua (" ") atau tanda kutip satu (' '). Jika tidak, Python akan mengira itu perintah rahasia dan akhirnya error.

💻 CONTOH KODE:

pesan = "Halo, nama saya Budi!" 
# Ini adalah String
angka = "12345" 
# (Ini juga String, karena diapit tanda kutip. Komputer menganggapnya teks biasa, bukan angka untuk dihitung).

📖 KATA 3: Integer (sering disingkat: int)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Angka bulat utuh. Tidak ada koma atau desimal.

💡 ANALOGI DUNIA NYATA:

Seperti menghitung barang yang tidak bisa dibelah. Misalnya jumlah manusia (1, 2, 3 orang), jumlah mobil, atau sisa nyawa di dalam game. Anda tidak mungkin bilang "jumlah manusia ada 2,5", bukan?

⚠️ ATURAN WAJIB:

Langsung ketik angkanya, jangan diapit tanda kutip. Jika diapit tanda kutip, komputer akan menganggapnya teks biasa (String), bukan angka matematika yang bisa dihitung.

💻 CONTOH KODE:

umur = 25 
# Ini adalah Integer, angka murni yang bisa dihitung/ditambah
umur = "25" 
# Ini BUKAN Integer, ini String teks biasa karena ada tanda kutip

📖 KATA 4: Float (dibaca: flot)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Angka desimal atau angka yang memiliki koma pecahan.

💡 ANALOGI DUNIA NYATA:

Seperti ukuran saat Anda menimbang berat badan, mengukur suhu tubuh, atau menghitung uang receh. Hasilnya tidak selalu bulat, bisa jadi 65,5 kg atau 36,7 derajat Celsius.

⚠️ ATURAN WAJIB:

Dalam Python, Anda wajib menggunakan titik ( . ), bukan koma ( , ) untuk menulis angka desimal.

💻 CONTOH KODE:

berat_badan = 65.5 
# Ini adalah Float yang benar
berat_badan = 65,5 
# Ini ERROR atau hasilnya jadi salah, karena dilarang memakai koma

📖 KATA 5: Boolean (dibaca: bu-li-an / sering disingkat: bool)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Data kepastian. Isinya BENAR-BENAR HANYA ADA DUA kemungkinan di dunia Python: Benar (True) atau Salah (False). Tidak ada kata ketiga, tidak ada kata "mungkin", "bisa jadi", atau "enggak tahu".

💡 ANALOGI DUNIA NYATA:

Persis seperti saklar lampu. Hanya ada dua pilihan absolut: Nyala atau Mati. Biasanya digunakan komputer untuk mengecek syarat, misalnya: Apakah user sudah bayar? Jawabannya cuma True (sudah) atau False (belum).

⚠️ ATURAN WAJIB:

Anda HANYA boleh mengetik kata True atau False. Huruf pertamanya wajib kapital dan tidak boleh memakai tanda kutip.

💻 CONTOH KODE:

sudah_lulus = True 
# Artinya: Benar, dia sudah lulus
sudah_bayar = False 
# Artinya: Salah, dia belum bayar
sudah_lulus = true 
# Ini ERROR karena huruf 't' nya kecil
sudah_lulus = "True" 
# (Ini SALAH, karena pakai tanda kutip, komputer akan menganggapnya teks biasa/String, bukan Boolean kepastian).

📖 KATA 6: List (dibaca: list)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Kumpulan banyak data yang disimpan berurutan di dalam satu tempat.

💡 ANALOGI DUNIA NYATA:

Bayangkan sebuah daftar belanjaan di selembar kertas. Di kertas itu tertulis berurutan: 1. Apel, 2. Jeruk, 3. Mangga. Ketimbang membuat banyak kotak (Variable) terpisah untuk masing-masing buah, lebih baik simpan semua sekaligus di satu daftar panjang.

⚠️ ATURAN WAJIB:

Seluruh isi daftar wajib diapit dengan kurung siku [ ] dan setiap isinya dipisahkan dengan tanda koma ( , ).

💻 CONTOH KODE:

keranjang_buah = ["Apel", "Jeruk", "Mangga"]

# Ini adalah satu List yang berisi tiga buah String

📖 KATA 7: Dictionary (dibaca: dik-sio-ne-ri)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Kumpulan data yang berpasangan. Ada "Kata Kunci" (Key) dan ada "Isinya" (Value).

💡 ANALOGI DUNIA NYATA:

Persis seperti buku kontak di HP Anda. Anda menyimpan nama (sebagai Kunci), dan menyimpan nomor telepon (sebagai Isinya). Saat butuh nomor Budi, Anda cukup mencari nama Budi.

⚠️ ATURAN WAJIB (BACA PELAN-PELAN):

  • 1. Wajib dibungkus dengan kurung kurawal { }.
  • 2. Kunci dan Isi dipisahkan oleh tanda titik dua ( : ).
  • 3. Jika Anda punya lebih dari satu pasang data, gunakan tanda koma ( , ) di tengah-tengah untuk memisahkan pasangan pertama dan pasangan berikutnya. (Kunci dan Isi bebas, tidak harus huruf dan angka, bisa berupa teks semua atau angka semua).

💻 CONTOH KODE:

kontak_hp = {"Budi": "0812345", "Andi": "0899999"}

# (Tanda koma di tengah itu berfungsi murni sebagai pemisah agar data si Budi dan data si Andi tidak saling bertabrakan).

📖 KATA 8: Tuple (dibaca: ta-pel)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Kumpulan data berurutan (mirip sebuah daftar), TAPI isinya dikunci mati oleh Python. Isinya sama sekali tidak bisa ditambah, dikurangi, atau diubah lagi oleh program.

💡 ANALOGI DUNIA NYATA:

Bayangkan menu makanan yang sudah dicetak permanen di buku menu restoran berbahan plastik tebal. Pelanggan dilarang keras mencoret, menghapus, atau mengubah daftar menu di buku tersebut.

⚠️ ATURAN WAJIB:

Wajib diapit dengan kurung biasa ( ).

💻 CONTOH KODE:

koordinat_bumi = (90.0, 180.0)

# (Karena ini memakai Tuple/kurung biasa, kalau nanti di tengah jalan Anda menyuruh komputer mengubah angkanya, komputer akan Error dan menolak keras).

📖 KATA 9: Index (dibaca: in-deks)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Nomor urut antrean dari sebuah data di dalam sebuah kelompok.

💡 ANALOGI DUNIA NYATA:

Seperti nomor kursi penonton di bioskop. TAPI HATI-HATI, komputer memiliki kebiasaan unik: komputer selalu mulai berhitung dari angka NOL (0), bukan angka 1. Jadi, barang yang posisinya paling depan, nomor Index-nya adalah 0.

⚠️ ATURAN WAJIB:

Untuk memanggil nomor urut, tulis nama wadahnya dan tempelkan kurung siku [ ] yang berisi angka urutannya.

💻 CONTOH KODE:

keranjang = ["Apel", "Jeruk", "Mangga"]

# Keterangan:
# Apel ada di Index 0
# Jeruk ada di Index 1
#Mangga ada di Index 2

📖 KATA 10: Comment (dibaca: komen / komentar)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Teks atau catatan yang sengaja ditulis oleh programmer di dalam kodingan hanya untuk dibaca oleh sesama manusia. Komputer akan tutup mata dan mengabaikan tulisan ini sepenuhnya.

💡 ANALOGI DUNIA NYATA:

Seperti sticky note (kertas tempel kecil) yang Anda tempelkan di pinggir dokumen resmi perusahaan untuk memberi pesan ke teman kerja ("Tolong bagian ini jangan dihapus ya!"). Kertas tempel itu bukan bagian dari isi dokumen resminya.

⚠️ ATURAN WAJIB:

Wajib diawali dengan tanda pagar ( # ). Apapun yang ditulis setelah tanda pagar pada baris tersebut tidak akan dianggap sebagai baris koding.

💻 CONTOH KODE:

# Ini adalah kode untuk menyimpan umur peserta
umur = 25

# (Kalimat yang diawali tanda # tidak akan membuat program error, karena komputer pura-pura tidak melihatnya).

📖 KATA 11: Syntax (dibaca: sin-taks)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Aturan mutlak soal tanda baca dan tata cara pengejaan saat kita mengetik kode.

💡 ANALOGI DUNIA NYATA:

Bayangkan Anda sedang menulis alamat email. Anda wajib memakai tanda keong (@). Kalau Anda menulis email tanpa tanda @, pasti emailnya gagal terkirim karena dianggap "tidak sah". Atau, saat Anda mengetik password, besar-kecil huruf sangat berpengaruh.

⚠️ ATURAN WAJIB:

Di dunia Python, jika Python meminta tanda kutip tapi Anda lupa menaruhnya, atau Python minta tanda titik dua ( : ) tapi Anda malah menaruh titik koma ( ; ), komputer akan langsung mogok kerja dan memunculkan peringatan Syntax Error (Salah Tanda Baca/Salah Ejaan).

💻 CONTOH KODE:

nama = "Budi

# (Ini akan langsung ERROR! Karena kurang satu tanda kutip penutup di belakang huruf "i". Kekurangan tanda baca sekecil inilah yang disebut pelanggaran Syntax).

📖 KATA 12: None (dibaca: nan)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Hampa. Tidak ada data sama sekali. Bukan nol (0), bukan teks kosong, tapi murni tidak ada isinya.

💡 ANALOGI DUNIA NYATA:

  • Angka 0 itu ibarat Anda punya dompet, dan di dalamnya ada koin bernilai nol. (Ada wujudnya: koin).
  • Teks kosong "" itu ibarat Anda punya kertas, tapi tidak ada tulisannya. (Ada wujudnya: kertas).
  • None itu ibarat Anda tidak punya dompet dan tidak punya kertas sama sekali. Vakum. Hampa udara. Biasanya digunakan komputer untuk memberi tahu, "Hei, kamu mencari data Budi, tapi datanya benar-benar tidak ada di sistem kami!"

⚠️ ATURAN WAJIB:

Huruf pertamanya wajib kapital (None), tidak boleh diberi tanda kutip, dan tidak bisa dihitung/ditambah dengan angka karena dia bukan angka.

💻 CONTOH KODE:

saldo_budi = None

# (Artinya Budi belum punya rekening sama sekali. Beda dengan saldo_budi = 0 yang artinya Budi punya rekening tapi sedang bokek).

BAB 2: KOMPOR & PISAU

Bab 2 / Kamus Python (Aksi & Output)

Di bab ini, kita tidak lagi sekadar menaruh data di dalam kotak (Variable). Kita akan menyuruh komputer melakukan sesuatu dengan data tersebut.

📖 KATA 13: print (dibaca: print)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Menampilkan sesuatu ke layar. Tanpa perintah ini, komputer sebenarnya berhitung di dalam otaknya, tapi kita sebagai manusia tidak akan bisa melihat hasilnya.

💡 ANALOGI DUNIA NYATA:

Bayangkan koki di dapur (komputer) sudah selesai memasak nasi goreng. Jika tidak ada pelayan yang mengantar nasi goreng itu ke meja tamu (print), maka si tamu tidak akan pernah melihat makanannya. print adalah sang pelayan.

⚠️ ATURAN WAJIB:

Wajib menggunakan huruf kecil semua dan diikuti kurung biasa ( ). Apapun yang ingin Anda tampilkan ke layar, taruh di dalam kurung tersebut.

💻 CONTOH KODE:

print("Halo Dunia!") 
# Layar komputer akan memunculkan tulisan: Halo Dunia!
print(umur) 
# (Layar akan memunculkan angka yang tersimpan di dalam kotak/variabel bernama umur).

📖 KATA 14: input (dibaca: in-put)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Meminta pengguna (user) mengetikkan sesuatu di keyboard, lalu komputer akan menangkap tulisan itu untuk diproses.

💡 ANALOGI DUNIA NYATA:

Seperti kasir kafe yang menyodorkan tablet dan bertanya, "Atas nama siapa Kak pesanannya?" Anda menjawab "Budi", lalu nama itu dicatat oleh kasir.

⚠️ ATURAN WAJIB:

Di dalam kurung ( ), Anda bisa menyelipkan teks pertanyaan (String) agar user tahu mereka harus mengetik apa. Perhatian: Apapun yang diketik oleh user (walaupun mereka mengetik angka), komputer akan selalu menangkapnya sebagai teks (String).

💻 CONTOH KODE:

nama_pelanggan = input("Siapa nama Anda? ")

# (Saat kode ini jalan, program akan berhenti sejenak, memunculkan pertanyaan "Siapa nama Anda?", dan menunggu Anda mengetik jawaban di keyboard).

📖 KATA 15: Function (dibaca: fang-syen / Fungsi)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Kumpulan langkah atau perintah kerja yang dibungkus menjadi satu nama, supaya tidak perlu diketik ulang capek-capek dari nol.

💡 ANALOGI DUNIA NYATA:

Seperti Buku SOP "Cara Bikin Kopi". Daripada Anda mengajari karyawan baru setiap hari: 1. Ambil gelas, 2. Masukkan kopi, 3. Tuang air panas, Anda cukup membuat satu SOP bernama bikin_kopi. Kapanpun Anda butuh kopi, Anda tinggal teriak: "Bikin kopi!", dan karyawan akan menjalankan ketiga langkah tadi secara otomatis.

⚠️ ATURAN WAJIB:

Untuk menciptakan fungsi baru, Anda wajib mengawalinya dengan kata sandi def (singkatan dari define / tetapkan), lalu ikuti dengan nama fungsinya, kurung biasa ( ), dan diakhiri titik dua ( : ).

💻 CONTOH KODE:

def bikin_kopi():
    print("Ambil gelas")
    print("Tuang air panas")

📖 KATA 16: Parameter (dibaca: pa-ra-me-ter)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Request khusus atau bahan tambahan yang Anda lempar ke dalam sebuah Function (Fungsi) agar hasilnya bisa disesuaikan.

💡 ANALOGI DUNIA NYATA:

Anda memanggil pelayan: "Tolong jalankan fungsi bikin_kopi, TAPI gulanya sedikit saja ya!". Nah, kalimat "gulanya sedikit" itu adalah Parameter. Jadi instruksinya sama, tapi hasil akhirnya bisa disesuaikan dengan permintaan.

⚠️ ATURAN WAJIB:

Parameter diletakkan di dalam kurung ( ) saat Anda membuat fungsi.

💻 CONTOH KODE:

def sapa_tamu(nama_tamu):
    print("Selamat datang, ", nama_tamu)

# (Di sini, nama_tamu adalah Parameter. Nanti saat program dijalankan, kita bisa mengganti nama_tamu dengan "Budi", "Andi", atau siapapun tanpa harus membuat fungsi baru).

📖 KATA 17: Return (dibaca: ri-tern)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Menyerahkan hasil akhir dari dalam Function (Fungsi) kembali ke tangan si peminta.

💡 ANALOGI DUNIA NYATA:

Anda menyuruh akuntan di dalam ruang tertutup: "Tolong hitung 5 ditambah 5".

  • Kalau akuntan itu pakai print, dia cuma akan teriak dari dalam kamar: "SEPULUHHH!" (Anda cuma dengar, tapi Anda tidak memegang angka itu untuk dihitung lagi).
  • Kalau akuntan itu pakai return, dia akan menulis angka 10 di selembar kertas, lalu menyerahkan kertas itu langsung ke tangan Anda. Anda bisa menyimpan kertas itu di saku Anda untuk dipakai jajan nanti.

⚠️ ATURAN WAJIB (BACA PELAN-PELAN):

  • 1. Ketik kata return diikuti dengan apa yang mau diserahkan (bisa berupa angka langsung, atau nama Variabel/kotak yang berisi hasil kerjanya).
  • 2. return adalah Garis Finish. Begitu komputer membaca baris yang ada kata return, pekerjaan langsung dihentikan (si akuntan keluar ruangan). Jangan menulis kode apapun lagi di bawah baris return karena komputer akan mengabaikannya.

💻 CONTOH KODE:

def tambah_angka():
    hasil = 5 + 5 
# Komputer menghitung 5+5 dan menaruh angka 10 di dalam kotak bernama 'hasil'
    return hasil 
# Komputer menyerahkan isi kotak 'hasil' kepada Anda
    print("Halo") 
# (Tulisan Halo ini TIDAK AKAN PERNAH muncul, karena posisinya ada di bawah 'return'. Komputer sudah berhenti bekerja).

📖 KATA 18: len (dibaca: len / singkatan dari kata bahasa Inggris Length yang berarti panjang)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Menghitung jumlah isi data.

💡 ANALOGI DUNIA NYATA:

Seperti satpam pemegang alat clicker di pintu masuk. Ia tidak peduli siapa namanya, ia cuma peduli jumlahnya.

⚠️ ATURAN WAJIB (BACA PELAN-PELAN):

Ketik len lalu ikuti dengan kurung biasa ( ). Cara komputer berhitung akan berubah otomatis tergantung jenis bungkusan yang Anda masukkan:

  • Jika yang di dalam kurung adalah teks/String (memakai tanda kutip " "), komputer akan menghitung jumlah hurufnya.
  • Jika yang di dalam kurung adalah daftar/List (memakai kurung siku [ ]), komputer akan berhenti menghitung huruf, dan beralih menghitung jumlah barang (item) yang dipisahkan oleh tanda koma.

💻 CONTOH KODE:

len("Budi") 
# (Komputer melihat tanda kutip, jadi dia menghitung huruf. Hasilnya: 4).
len(["Apel", "Jeruk", "Mangga"]) 
# (Komputer melihat kurung siku, jadi dia menghitung jumlah barang. Hasilnya: 3).

    print(angka)

📖 KATA 40: return (dibaca: ri-tern / artinya: Kembalikan)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Instruksi kepada komputer untuk memberikan hasil akhir dari resep yang sudah dia kerjakan, agar hasil itu bisa kita simpan ke dalam wadah (Variabel) atau kita olah lagi.

uang_tamu = 1000 # Variabel Global (Bisa dipakai siapa saja)

BAB 7: KOTAK P3K DAPUR

Bab 7 / Kamus Python (Penanganan Error / Exception Handling)

Saat belajar memasak, pasti ada kalanya masakan gosong atau Anda tanpa sengaja menumpahkan minyak. Di Python, hal tersebut disebut sebagai Error. Daripada dapur Anda kebakaran (program mati total), kita bisa menyiapkan "Kotak P3K" untuk menangani tumpahan minyak tersebut dengan tenang.

📖 KATA 48: try (dibaca: trai / artinya: Coba lakukan)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

"Komputer, aku tahu instruksi ini agak berisiko dan mungkin bisa gagal. Jadi, coba saja dulu secara perlahan. Kalau gagal, jangan langsung marah dan mematikan program, ya!"

💡 ANALOGI DUNIA NYATA:

Seperti Anda sedang menyuruh anak kecil belajar memecahkan telur di atas wajan. Anda bilang, "Coba (try) pecahkan telurnya ke wajan. Kalau ternyata cangkangnya ikut jatuh, tidak apa-apa, jangan menangis (jangan crash)."

⚠️ ATURAN WAJIB:

Perintah try tidak bisa berdiri sendiri. Ia wajib dipasangkan dengan perintah except di bawahnya.

💻 CONTOH KODE:

try:
    # Komputer mencoba membagi angka dengan NOL (Ini adalah tindakan ilegal di matematika!)

📖 KATA 49: except (dibaca: ek-sep / artinya: Kecuali / Tangkap)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Rencana B (Rencana Cadangan). "Kalau instruksi di dalam kotak try tadi benar-benar gagal/error, segera tangkap error-nya dan jalankan instruksi penyelamatan di bawah ini!"

💡 ANALOGI DUNIA NYATA:

Menyambung analogi telur tadi.
Coba pecahkan telurnya ke wajan (try). Jika gagal (except) karena cangkangnya ikut jatuh, segera ambil sendok dan buang cangkangnya. Ini adalah rencana cadangan agar proses memasak tetap bisa dilanjutkan.

⚠️ ATURAN WAJIB:

Letakkan di bawah blok try. Pastikan Anda memajukan teks (Tab) untuk instruksi di dalamnya.

💻 CONTOH KODE:

try:
    hasil = 10 / 0
    print(hasil)
except:
    # Karena pembagian nol itu error, komputer melompat ke baris ini:
    print("Waduh Bos, angka tidak bisa dibagi dengan nol!")

📖 KATA 50: finally (dibaca: fai-ne-li / artinya: Akhirnya / Bagaimanapun juga)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

Instruksi mutlak sang "tukang bersih-bersih". "Komputer, tidak peduli apakah proses tadi berhasil dengan mulus atau berujung gagal/error, pokoknya kamu wajib menjalankan instruksi terakhir ini sebelum pergi!"

💡 ANALOGI DUNIA NYATA:

Anda selesai memasak. Entah masakannya enak (berhasil) atau gosong total (error), ada satu aturan mutlak di dapur: "Bagaimanapun juga (finally), kompor harus dimatikan dan dapur harus disapu!"

Di dunia koding, finally sering dipakai untuk memastikan komputer "menutup koneksi database" atau "menutup file" agar memori laptop tidak penuh, terlepas dari programnya error atau tidak.

⚠️ ATURAN WAJIB:

Diletakkan paling bawah, setelah try dan except.

💻 CONTOH KODE:

try:
    print("Sedang merebus air...")
    # Anggap ada error disini
except:
    print("Gagal merebus air!")
finally:
    print("Matikan kompor!") # Baris ini AKAN SELALU dicetak, apapun yang terjadi di atas.

📖 KATA 51: else (dalam blok try-except) (dibaca: els / artinya: Jika Tidak Ada Masalah)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

"Komputer, ini hadiah atau langkah ekstra yang HANYA boleh kamu lakukan JIKA instruksi di dalam try tadi sukses 100% tanpa ada error sedikitpun."

💡 ANALOGI DUNIA NYATA:

Kembali ke analogi memasak telur.
Anda bilang: "Coba (try) pecahkan telur ke wajan. Jika cangkangnya jatuh (except), buang cangkangnya. TAPI JIKA SEMUANYA LANCAR (else) tanpa ada cangkang yang jatuh, taburkan garam dan lada ke atas telur itu!"

Langkah menaburkan garam (else) ini hanya masuk akal dilakukan kalau telurnya berhasil dipecahkan dengan sempurna. Kalau telurnya gagal/tumpah, menaburkan garam jadi tidak ada gunanya, bukan?

⚠️ ATURAN WAJIB:

Harus diletakkan setelah blok except dan sebelum blok finally (jika ada).

💻 CONTOH KODE:

try:
    # Komputer mencoba membagi angka yang benar
    hasil = 10 / 2
except:
    print("Ada yang salah dengan perhitunganmu!")
else:
    # Karena pembagian di atas berhasil (tidak error), baris ini dijalankan:

📖 KATA 52: except Exception as e (dibaca: ek-sep ek-sep-syen es i / artinya: Tangkap dan Baca Laporannya)

🗣️ ARTINYA DALAM BAHASA MANUSIA:

"Komputer, kalau ada error, tolong jangan cuma bilang 'Gagal!', tapi tangkap pesan asli dari error tersebut, masukkan ke dalam amplop berlabel e, dan bacakan ke aku agar aku tahu penyebab pastinya!"

💡 ANALOGI DUNIA NYATA:

Bayangkan paket belanjaan Anda gagal dikirim. Kalau kurir cuma bilang "Paket gagal dikirim", Anda pasti bingung (apakah karena alamat salah, rumah kosong, atau tersasar?). Tapi dengan Exception as e, kurir itu memberi Anda surat resminya: "Paket gagal dikirim KARENA (e) rumah Anda terkunci dan digembok."

⚠️ ATURAN WAJIB:

Huruf e sebenarnya bebas diganti apa saja (seperti as error atau as masalah), tapi 99% programmer sedunia sepakat menggunakan huruf e sebagai singkatan universal dari error atau exception.

💻 CONTOH KODE:

try:

Artificial Intelligence (AI)

Kecerdasan buatan. Kemampuan komputer untuk meniru cara berpikir, belajar, dan mengambil keputusan layaknya manusia.

Machine Learning (ML)

Cabang AI di mana mesin belajar sendiri dari data tanpa perlu diprogram aturan pastinya satu per satu oleh manusia.

Deep Learning (DL)

Cabang ML yang lebih canggih, menggunakan jaringan saraf tiruan berlapis-lapis untuk memproses data rumit seperti gambar, suara, atau teks panjang.

Supervised Learning

Mesin belajar dari sekumpulan data yang sudah diberi "kunci jawaban" (label) oleh manusia.

Unsupervised Learning

Mesin dibiarkan mencari sendiri pola tersembunyi dari sekumpulan data mentah yang tidak memiliki kunci jawaban.

Reinforcement Learning

Mesin belajar melalui sistem "coba-coba" dengan diberi hadiah (reward) jika benar dan hukuman (penalty) jika salah.

Neural Network

Jaringan saraf tiruan. Algoritma komputer yang terinspirasi dari cara kerja sel otak manusia yang saling terhubung.

Training

Proses "sekolah". Masa di mana model AI diberi makan banyak data agar ia belajar dan menjadi pintar.

Inference

Proses "ujian". Momen di mana model AI yang sudah pintar disuruh memprediksi data baru yang belum pernah ia lihat sebelumnya.

Overfitting

Kondisi saat AI terlalu "menghafal" data latihan sehingga ia mendapat nilai sempurna di kelas, tapi gagal total saat diuji di dunia nyata.

Underfitting

Kondisi saat AI terlalu bodoh (kurang belajar), sehingga tidak bisa memahami pola data sama sekali.

Feature

Atribut atau petunjuk dari sebuah data. (Contoh: Untuk menebak harga rumah, feature-nya adalah luas tanah dan jumlah kamar).

Label / Target

Jawaban akhir yang ingin ditebak oleh AI. (Contoh: Harga rumah itu sendiri).

Epoch (dibaca: e-pok)

Satu putaran penuh di mana AI selesai membaca seluruh data latihan dari awal sampai akhir.

Batch Size

Jumlah baris data yang dibaca AI sekaligus dalam satu waktu sebelum ia memperbarui ingatannya.

Accuracy (Akurasi)

Persentase seberapa sering tebakan AI itu benar secara keseluruhan.

Precision

Seberapa akurat AI saat ia menebak "Iya/Positif". (Metrik untuk menghindari salah tuduh).

Recall

Seberapa jago AI menemukan semua kasus positif yang tersembunyi. (Metrik untuk menghindari ada data yang terlewat).

F1-Score

Nilai keseimbangan atau nilai tengah antara Precision dan Recall.

Loss Function

Alat ukur untuk menghitung "seberapa melenceng" tebakan AI dari jawaban aslinya. Semakin kecil angkanya, semakin bagus.

Gradient Descent

Cara AI memperbaiki dirinya sendiri dengan melangkah turun mencari titik kesalahan (Loss) paling rendah.

Hyperparameter

Pengaturan manual (seperti memutar tombol radio) yang harus diatur oleh Engineer sebelum AI mulai belajar

Fine-Tuning

Memoles model AI raksasa yang sudah pintar dengan sedikit data khusus agar ia menjadi ahli di satu bidang spesifik (misal: AI ahli hukum).

Model (Model AI)

"Otak" digital dari sebuah AI yang sudah selesai diajari untuk melakukan tugas tertentu (seperti menjawab pertanyaan, menerjemahkan bahasa, atau mengenali gambar).

Dataset

Kumpulan data mentah (bisa berupa jutaan teks, tabel, atau dokumen) yang digunakan sebagai "buku pelajaran" untuk memberi makan dan melatih AI.

Algoritma (Algorithm)

Urutan langkah-langkah logis untuk menyelesaikan suatu masalah. Ibarat resep masakan dari tahap persiapan hingga makanan siap dihidangkan.

Data Pipeline

Jalur pipa digital otomatis tempat data mengalir, disaring, dan dirapikan dari sumber mentah hingga siap disuapkan ke AI

Structured Data

Data yang sangat rapi dan punya kolom/baris yang jelas (seperti file Excel atau tabel database)

Unstructured Data

Data yang berantakan, bebas, dan tidak punya format baku (seperti isi email, artikel PDF, gambar, atau video).

🧠 KATEGORI 2: GENERATIVE AI, LLM & PROMPTING

(Kategori ini mencakup teknik-teknik lanjutan dalam membangun dan mengoptimalkan sistem AI generatif seperti RAG)

Generative AI (GenAI)

Jenis AI yang bukan cuma bisa menganalisis data, tapi bisa "menciptakan" hal baru dari nol (teks, gambar, suara, kode).

Natural Language Processing (NLP)

Cabang AI yang mengajari komputer untuk bisa membaca, memahami, dan membalas bahasa manusia, bukan sekadar bahasa pemrograman.

Foundation Model

Model raksasa serbaguna yang sudah dilatih dengan porsi besar data dari seluruh internet. Ibarat orang pintar yang tahu banyak hal umum, tapi belum dilatih untuk satu spesialisasi.

Parameters

"Jutaan tombol sakelar" di dalam otak model AI. Semakin banyak jumlah parameternya (seperti 7B atau 70B), biasanya modelnya semakin pintar dan kompleks, tapi butuh komputer yang lebih besar.

Weights (Bobot)

Nilai kekuatan dari hubungan antar neuron di otak AI. Bobot inilah yang berubah-ubah saat AI sedang belajar.

Prompt Engineering

Seni menyusun kalimat perintah (prompt) yang detail dan spesifik, agar AI memberikan jawaban yang tepat tanpa halusinasi.

System Prompt

Perintah rahasia di belakang layar yang mengatur kepribadian atau batasan AI sebelum ia mengobrol dengan user. (Misal: "Kamu adalah agen layanan pelanggan yang ramah. Jangan pernah bahas politik").

User Prompt

Pertanyaan atau instruksi mentah yang diketik langsung oleh pengguna aplikasi.

Zero-Shot Prompting

Meminta AI mengerjakan tugas baru tanpa memberinya contoh sama sekali.

Few-Shot Prompting

Memberikan satu atau beberapa contoh (tanya-jawab) terlebih dahulu di dalam prompt, agar AI paham format yang kita inginkan sebelum dia menjawab soal yang sebenarnya.

Chain of Thought (CoT)

Teknik prompting yang memaksa AI untuk "berpikir langkah demi langkah" (menjabarkan proses berpikirnya) sebelum memberikan jawaban akhir. Sangat ampuh untuk tugas logika dan matematik

Hallucination (Halusinasi)

Momen ketika AI dengan sangat percaya diri memberikan jawaban yang salah, ngarang, atau tidak masuk akal.

Context Window

Batas maksimal ingatan jangka pendek AI dalam satu percakapan. (Misal: Batas 8000 token berarti AI akan "lupa" dengan obrolan di awal jika percakapannya sudah melebihi batas tersebut).

Temperature

Pengatur tingkat kreativitas AI. Angka mendekati 0 membuat AI menjawab kaku dan kaku (pasti). Angka mendekati 1 membuat AI sangat kreatif (kadang ngelantur)

Top-K / Top-P

Pengaturan lanjutan bersama Temperature untuk membatasi opsi kata-kata aneh yang boleh dipilih AI saat menyusun kalimat.

Embedding

Proses mengubah teks (kata/kalimat) menjadi deretan angka (vektor koordinat). Tujuannya agar mesin bisa menghitung seberapa mirip makna antara dua kalimat.

Vector Database

Gudang khusus (seperti Pinecone atau FAISS) untuk menyimpan angka-angka Embedding. Beda dengan database biasa yang mencari kecocokan huruf, Vector DB mencari "kecocokan makna".

Chunking

Proses memotong-motong dokumen panjang (seperti buku PDF) menjadi paragraf-paragraf kecil agar muat dimasukkan ke dalam ingatan AI (Context Window).

Cosine Similarity

Rumus matematika yang dipakai Vector Database untuk mengukur seberapa mirip dua potong teks berdasarkan arah koordinat angkanya.

Agent (Agen Otonom)

Sistem AI yang tidak hanya menunggu pertanyaan, tapi bisa merencanakan langkah, memilih alat bantu (tools), dan mengambil keputusan sendiri untuk mencapai tujuan.

Tool Calling / Function Calling

Kemampuan LLM untuk menekan tombol "eksekusi alat luar" (seperti mencari di Google atau menarik data Excel) jika dia merasa butuh info tambahan.

LangChain

Pustaka (Library) paling populer untuk merangkai LLM, Vector DB, dan Prompt menjadi satu aplikasi utuh (rantai logika).

LangGraph

Kerangka kerja khusus (dari pembuat LangChain) untuk merancang alur kerja agen otonom yang bisa berpikir berulang-ulang tanpa batas (siklik/looping).

OpenAI API

Layanan berbayar untuk meminjam otak AI canggih milik OpenAI (seperti GPT-4) ke dalam kodingan kita.

Hugging Face

"GitHub-nya dunia AI". Website tempat orang-orang dari seluruh dunia menyimpan, berbagi, dan mengunduh ribuan model AI secara gratis.

Token

Hitungan "potongan kata" yang dibaca oleh AI. Semakin panjang Anda mengobrol dengan AI, semakin banyak token yang dikuras.

LLM (Large Language Model)

Jenis Model AI raksasa yang dikhususkan hanya untuk memahami dan memproduksi teks bahasa manusia (contohnya ChatGPT, Gemini, atau Claude).

RAG (Retrieval-Augmented Generation)

Teknik memberi "buku contekan" kepada AI. Daripada AI menjawab dari hafalannya sendiri (yang sering halusinasi/ngarang), RAG memaksa AI membaca dokumen rahasia kita dulu sebelum menjawab.

Hybrid Search

Teknik pencarian tingkat tinggi di RAG yang menggabungkan kecocokan kata persis (berdasarkan huruf) dan kecocokan makna (berdasarkan vektor).

Re-ranking / Cross Encoder

Tahap "menyortir ulang". Setelah sistem RAG menemukan 10 dokumen, AI khusus akan mengurutkan ulang mana dokumen yang paling relevan untuk ditaruh di urutan nomor 1

Semantic Cache

"Ingatan pintar". Sistem yang menyimpan jawaban AI agar jika ada pengguna lain menanyakan hal yang maknanya sama, sistem tidak perlu membayar API lagi untuk berpikir ulang.

Quantization (Kuantisasi)

Teknik "mengompres" ukuran model AI raksasa agar muat dijalankan di laptop atau server biasa tanpa terlalu mengorbankan kepintarannya.

PEFT (Parameter-Efficient Fine-Tuning)

Melatih ulang model AI raksasa dengan cara hemat, yaitu hanya mengutak-atik sebagian kecil sel otaknya saja.

LoRA (Low-Rank Adaptation)

Salah satu teknik PEFT paling populer untuk membuat AI menjadi "spesialis" di bidang tertentu hanya dengan modal komputasi (GPU) yang kecil.

Streaming (LLM Streaming)

Efek mesin ketik. Menampilkan jawaban AI kata-demi-kata secara real-time di layar pengguna tanpa harus menunggu AI selesai memikirkan seluruh paragraf.

Agent Memory

Kemampuan sebuah sistem AI otonom untuk merekam dan mengingat riwayat percakapan dengan Anda dari minggu-minggu sebelumnya.

TTS (Text-to-Speech) & STT (Speech-to-Text)

Teknologi yang mengubah teks menjadi suara (TTS) atau suara menjadi teks (STT), memungkinkan AI untuk berbicara dan mendengar.

OCR (Optical Character Recognition)

Teknologi "mata pembaca" untuk mengekstrak tulisan yang terkunci di dalam format gambar atau hasil scan PDF.

Open-Weights

Model AI yang "otaknya" bisa diunduh gratis, tapi kode resep pelatihannya tidak dibongkar ke publik.

Proprietary Model (Closed Source)

Model AI tertutup, rahasia, dan berbayar yang tidak bisa kita unduh ke laptop (hanya bisa diakses lewat API), seperti ChatGPT atau Gemini Advanced.

Guardrails

Pagar pengaman otomatis. Sistem pelindung untuk mencegat pengguna yang iseng bertanya hal aneh, atau mencegat AI yang ingin merespons dengan kata-kata tidak pantas.

Prompt Injection

Serangan hacker gaya baru, di mana pengguna menyelundupkan teks jahat ke dalam prompt agar AI melanggar aturan awal pembuatnya.

Jailbreaking

Usaha menjebol "otak" AI agar ia mau menjawab hal-hal ilegal yang seharusnya dilarang.

☁️ KATEGORI 3: INFRASTRUKTUR, CLOUD, & DEPLOYMENT

(Kategori ini fokus pada server, hardware, arsitektur backend, dan manajemen API)

Deployment

Proses memindahkan kode aplikasi dari komputer pribadi (lokal) ke server di internet agar bisa diakses oleh orang lain secara global.

Local

Lokal berarti program Anda baru bisa berjalan dengan aman di laptop Anda sendiri.

Production (Prod)

Lingkungan "medan tempur" asli. Aplikasi Anda sudah rilis dan sedang digunakan oleh pengguna sungguhan, bukan lagi untuk uji coba.

Staging / Development

Lingkungan "latihan". Tempat di mana kode diuji coba untuk mencari bug sebelum dirilis ke lingkungan Production.

Container (Docker)

Kotak kedap udara berisi aplikasi Anda beserta semua alat pendukungnya. Tujuannya agar aplikasi jalan lancar di komputer manapun, tanpa takut "beda settingan".

Image (Docker Image)

"Cetak biru" atau blueprint dari Container. Sebelum Container dijalankan, ia harus dibuat dari sebuah Image.

Container Registry

Gudang penyimpanan khusus untuk menyimpan berbagai Image Docker (seperti Docker Hub atau Google Artifact Registry).

API (Application Programming Interface)

Jembatan komunikasi atau "Pelayan Pembawa Pesan". Misalnya, kode di laptop Anda menyuruh API pergi ke server Google untuk meminta jawaban AI, lalu API membawa jawaban itu kembali ke layar Anda.

API Gateway

"Gerbang masuk" utama untuk semua permintaan dari pengguna sebelum diteruskan ke backend AI Anda. Berfungsi mengatur lalu lintas dan keamanan.

REST API

Standar aturan (tata krama) paling umum di dunia web yang dipakai agar dua aplikasi berbeda bisa saling bertukar pesan di internet.

Load Balancer

Pengatur lalu lintas otomatis. Jika ada ribuan orang mengakses AI Anda sekaligus, alat ini membagi beban kerja ke beberapa server agar sistem tidak crash.

Latency (Latensi)

Jeda waktu tunggu. Waktu yang dibutuhkan AI untuk memproses pertanyaan hingga mengirimkan jawaban. (Semakin kecil nilainya, semakin terasa instan).

Throughput

Kapasitas maksimal. Seberapa banyak permintaan atau percakapan yang bisa ditangani oleh sistem Anda dalam satu detik.

Scaling

Proses menambah kekuatan sistem. Horizontal scaling (tambah server) atau Vertical scaling (tambah spek server).

Managed Service

Layanan di cloud di mana Anda cukup menyewa/memakai layanannya saja, tanpa perlu pusing mengurus infrastruktur fisiknya (seperti Cloud Run atau AWS Lambda).

Serverless

Paradigma di mana Anda tidak perlu memikirkan server. Kode Anda hanya aktif (dan membayar) saat ada permintaan masuk.

CI/CD (Continuous Integration/Continuous Deployment)

Jalur otomatis untuk mengetes dan mengirim kode baru ke Production tanpa campur tangan manusia manual.

Endpoint

URL atau alamat tujuan spesifik di internet tempat API Anda "menunggu" permintaan datang.

Environment Variables (.env)

Tempat menyimpan data rahasia (seperti API Key atau password database) agar tidak pernah tertulis langsung di dalam kode utama.

Logs

Catatan aktivitas sistem yang dicatat setiap detik. Jika ada error, log adalah "kotak hitam" pesawat yang memberi tahu apa yang terjadi sesaat sebelum sistem mati.

Monitoring

Memantau kesehatan sistem secara real-time (seperti detak jantung) untuk mendeteksi tanda-tanda sistem akan down.

Infrastructure as Code (IaC)

Menulis kode untuk membangun infrastruktur server/cloud. Jadi, membangun server cukup pakai script, bukan klik-klik manual di dashboard.

SSH (Secure Shell)

Cara aman untuk meremote atau mengendalikan komputer server dari jarak jauh hanya dengan teks perintah.

SSL/TLS

Lapisan keamanan yang membuat koneksi antara user dan server Anda terenkripsi (logo gembok di browser).

Server

Komputer super yang menyala 24 jam tanpa henti untuk menaruh aplikasi kita agar selalu bisa diakses orang lain lewat internet.

Database (DB)

Gudang digital raksasa tempat menyimpan semua data (seperti nama user, riwayat chat, atau dokumen) secara permanen.

Cloud (Komputasi Awan)

Menyewa komputer super canggih milik perusahaan besar (seperti Google, Amazon, atau Microsoft) lewat internet, sehingga kita tidak perlu membeli komputer/server fisik yang mahal.

GPU (Graphics Processing Unit)

Mesin pemroses grafis yang menjadi "mesin turbo" wajib untuk bisa menjalankan dan melatih model AI dengan cepat.

VRAM (Video RAM)

Kapasitas memori khusus di dalam GPU. Semakin besar angka VRAM-nya, semakin besar ukuran model AI yang bisa ditampung.

Webhook

"Kurir instan". Sistem yang otomatis mengetuk pintu aplikasi Anda dan mengirimkan data begitu ada sebuah kejadian selesai diproses di server lain.

Rate Limit (Error 429)

"Tilang" dari penyedia API karena aplikasi Anda mengirim terlalu banyak permintaan obrolan dalam hitungan detik/menit.

⚙️ KATEGORI 4: WORKFLOW, SOFTSKILL, & LAINNYA

(Kategori ini mencakup metodologi pengembangan software secara umum)

Agile

Metode kerja cepat yang membagi proyek besar menjadi bagian-bagian kecil (sprint) agar bisa selesai dengan cepat dan mudah diubah.

Sprint

Periode waktu singkat (biasanya 1-2 minggu) untuk menyelesaikan tugas-tugas spesifik dalam proyek.

Scrum

Kerangka kerja Agile yang memiliki rutinitas harian (Daily Standup) untuk memastikan tim tetap sinkron.

Kanban

Sistem manajemen tugas menggunakan papan visual (seperti Trello/Jira) untuk melihat status tugas (To Do, In Progress, Done).

Code Review

Proses di mana sesama anggota tim saling memeriksa kode satu sama lain sebelum digabungkan ke proyek utama.

Pull Request (PR)

Cara mengajukan perubahan kode agar diperiksa oleh anggota tim lain sebelum disetujui.

Merge

Menggabungkan kode yang sudah diperiksa ke dalam proyek utama.

Branch

"Cabang" kode. Tempat kita bereksperimen mengubah fitur tanpa merusak kode utama yang sedang berjalan.

Conflict

Momen saat dua orang mengubah baris kode yang sama di waktu bersamaan, sehingga komputer bingung mana yang harus dipakai.

Documentation

Catatan manual (dalam bentuk teks atau README) yang menjelaskan cara kerja sistem agar orang lain bisa menggunakannya.

Technical Debt

"Utang teknis". Kode yang dibuat asal-asalan demi cepat selesai, namun nantinya harus diperbaiki agar tidak menjadi masalah besar.

MVP (Minimum Viable Product)

Versi paling sederhana dari sebuah aplikasi yang sudah bisa berfungsi dan layak diberikan ke klien.

Prototype

Model awal yang dibuat kasar untuk menguji apakah ide tersebut bisa berjalan atau tidak.

Stakeholder

Pihak yang berkepentingan dalam proyek (bisa klien, bos, atau pengguna akhir).

Requirement

Kebutuhan fitur yang harus ada di dalam aplikasi berdasarkan permintaan klien.

Bug Bounty

Program di mana perusahaan memberi hadiah uang kepada siapapun yang menemukan celah keamanan di sistem mereka.

Compliance

Kepatuhan terhadap aturan hukum atau standar privasi data (misalnya GDPR atau aturan perlindungan data lokal).

Scalability

Kemampuan sistem untuk tetap stabil saat jumlah pengguna melonjak drastis.

Maintenance

Pekerjaan rutin untuk menjaga sistem tetap nyala, memperbaiki error baru, atau memperbarui fitur.

Unit Test

Kodingan khusus yang tugasnya mengetes bagian terkecil dari aplikasi Anda secara otomatis.

Production-Ready

Status kode yang dianggap sudah cukup aman, stabil, dan teruji untuk dijalankan di dunia nyata.

Documentation-First

Pendekatan di mana Anda menulis dokumentasi sebelum menulis kode.

Open-Source License

Aturan hukum yang membolehkan atau melarang orang lain untuk menggunakan kode Anda (seperti MIT, Apache 2.0).

Soft Skill

Kemampuan non-teknis, seperti komunikasi, manajemen waktu, dan empati saat menghadapi konflik tim.

Pair Programming

Dua orang bekerja di depan satu komputer: satu menulis kode, satu lagi mengawasi dan memberikan ide.

Refactoring

Merapikan kode tanpa mengubah hasilnya.

Backlog

Daftar panjang semua fitur yang ingin dibuat, tapi belum dikerjakan saat ini.

Issue Tracking

Sistem untuk mencatat setiap bug atau fitur baru agar tidak terlupa (di GitHub disebut Issues).

Release Cycle

Jadwal rutin kapan aplikasi akan diperbarui (misal: dirilis versi baru setiap bulan).

Versioning

Cara menandai versi aplikasi (misal: v1.0.0, v1.0.1) agar pengguna tahu apakah aplikasi tersebut baru atau sudah lama.

Alpha/Beta Testing

Tahap uji coba aplikasi: Alpha (internal), Beta (ke beberapa pengguna terpilih).

Documentation (Docstring)

Komentar singkat di dalam kode yang menjelaskan apa fungsi dari sebuah blok kode.

ReadMe

File teks paling penting di sebuah folder proyek yang menjelaskan tentang apa proyek itu dan bagaimana cara pakainya.

Dependencies

Daftar Library lain yang harus terinstal agar kode Anda bisa berjalan.

Environment (Env)

Ruangan kerja terisolasi di dalam laptop. Tujuannya agar alat-alat masak (Library) untuk Proyek A tidak tercampur dan merusak alat-alat di Proyek B. (Catatan: merevisi istilah Env sebelumnya agar lebih general).

Virtual Environment (venv / conda)

Ruangan kerja terisolasi di dalam laptop. Tujuannya agar alat-alat masak (Library) untuk Proyek A tidak tercampur dan merusak alat-alat di Proyek B. (Catatan: merevisi istilah Env sebelumnya agar lebih general).

Debugging Tools

Alat bantu untuk melacak bug (misal: debugger di VS Code).

Architecture Pattern

Pola baku dalam merancang sistem (misal: Model-View-Controller).

Cross-Platform

Aplikasi yang bisa jalan di Windows, Mac, maupun Linux tanpa perlu diubah kodenya.

User Experience (UX)

Seberapa nyaman dan gampang orang menggunakan sistem Anda.

User Interface (UI)

Tampilan visual aplikasi Anda.

Ngoding / Coding

Kegiatan mengetik instruksi (resep) untuk komputer.

Run / Execute

Tombol sakelar untuk menyuruh komputer "Ayo jalan! Kerjakan tugas yang baru saja saya ketik!".

Bug (dibaca: Bag)

Harafiahnya berarti Serangga/Kutu. Di dunia IT, ini artinya ada "kecacatan" atau kesalahan logika di dalam kode Anda yang membuat program berjalan tidak semestinya.

Debugging (dibaca: Di-ba-ging)

Kegiatan mencari dan membasmi si "Kutu" (Bug) tadi sampai programnya lancar kembali. Kegiatan ini memakan 80% waktu seorang programmer.

Library / Package

"Kotak peralatan ekstra" buatan orang lain yang bisa Anda pinjam ke dalam kode Anda (lewat perintah import), agar Anda tidak perlu membuat semuanya dari nol.

Terminal / CLI / Command Prompt

Layar hitam berisi teks putih tempat Anda mengetikkan perintah langsung ke "jantung" sistem komputer.

Repo / Repository

Gudang penyimpanan digital (biasanya menggunakan website GitHub) untuk menyimpan seluruh file kodingan Anda agar aman dan bisa dikerjakan bersama tim.

Frontend (Front-end)

Bagian "Etalase". Semua tampilan seperti tombol, warna, dan teks di aplikasi yang bisa dilihat langsung oleh pengguna.

Backend (Back-end)

Bagian "Dapur". Mesin tak terlihat di belakang layar yang sibuk menghitung data dan berpikir.

Open Source

Program gratisan yang "resep" kodingannya dibuka untuk umum. Siapa saja boleh melihat, mencontek, atau memodifikasinya tanpa perlu membayar lisensi.

Framework

Kerangka kerja setengah jadi. Ibarat membangun rumah, pondasi dan tiangnya sudah dibikinkan oleh orang lain, Anda tinggal mengisi temboknya saja agar lebih cepat selesai.

Git & GitHub

Git adalah alat "mesin waktu" untuk merekam setiap perubahan pada kodingan Anda agar bisa dikembalikan jika rusak. Sedangkan GitHub adalah semacam Google Drive khusus programmer untuk menyimpan dan memamerkan kode tersebut ke internet.

IDE / Code Editor

Perangkat lunak atau aplikasi tempat Anda mengetik, mengedit, dan menjalankan kode (contoh: Visual Studio Code, PyCharm).

PIP (Python Package Installer)

"Toko aplikasi" bawaan Python. Alat ini bertugas untuk mengunduh dan memasang Library tambahan dari internet ke dalam laptop Anda.

Hardcode (Hard-coding)

Praktik menuliskan data secara permanen langsung ke dalam kode. (Contoh: Menulis nama "Budi" di dalam kode, ketimbang menyuruh komputer menanyakannya lewat input()).

Clean Code

Praktik menulis kodingan yang rapi, terstruktur, dan mudah dipahami oleh manusia (programmer lain), bukan hanya sekadar bisa dibaca oleh komputer.

Script / Scripting

Sebuah file tunggal (biasanya berakhiran .py) yang berisi rentetan perintah kode Python untuk melakukan satu tugas otomatis.

Web Scraping

Teknik membuat program Python yang bisa menyedot, menyalin, dan mengumpulkan data dari suatu halaman website secara otomatis.

OOP (Object-Oriented Programming)

Gaya penulisan kode tingkat lanjut yang menganggap komponen di dalam program sebagai "Benda/Objek" dunia nyata (misal: objek "Mobil" punya warna dan punya mesin).

Path / Directory

Alamat pasti lokasi sebuah file atau folder di dalam sistem komputer Anda (contoh: C:/Dokumen/proyek_python/).

Syntax Sugar

Cara penulisan kode di Python yang dirancang sangat singkat dan manis (mudah dibaca), untuk menggantikan cara penulisan lama yang panjang dan rumit.

PEP 8 (dibaca: pep-eight)

Buku pedoman resmi tata krama penulisan Python sedunia. Kalau ada yang bilang "Kodinganmu gak PEP 8 nih", artinya kodingan Anda berantakan dan tidak mengikuti standar kerapian internasional.

JSON (JavaScript Object Notation)

Format teks universal untuk tukar-menukar data di internet. Bentuknya 99% mirip dengan Dictionary di Python (memakai kurung kurawal { }). Anda pasti akan bertemu ini saat meminjam data dari API.

Refactoring (Refaktor)

Kegiatan "bersih-bersih bengkel". Merapikan dan menyederhanakan kode yang sudah jalan agar lebih cepat dan enak dibaca, tanpa mengubah fungsi atau hasil akhirnya.

Try-Except (Error Handling)

Sabuk pengaman dalam coding. Daripada program tiba-tiba mati total ( crash ) saat kena error, kita menyuruh komputer: "Coba (Try) jalankan ini, kalau gagal/error, Tangkap (Except) dan jalankan rencana cadangan".

Boilerplate

Teks "Kop Surat" atau template kode standar yang panjang dan terpaksa harus diketik berulang-ulang setiap kali membuat proyek baru, sebelum kita benar-benar menulis kode inti kita.

PoC (Proof of Concept)

Aplikasi contoh atau model uji coba berskala kecil yang dibuat dengan cepat untuk membuktikan apakah suatu ide atau konsep teknologi benar-benar bisa diwujudkan sebelum proyek besar dimulai.

Microservices

Arsitektur memecah satu aplikasi raksasa menjadi puluhan aplikasi kecil yang hidup mandiri (misal: bagian khusus chat terpisah dari bagian database).

Monolith

Arsitektur jadul di mana seluruh fungsi aplikasi (layar, logika, dan database) ditumpuk ke dalam satu program raksasa.

🔴 PERINGATAN ERROR

Menerjemahkan Teks Merah Menjadi Bahasa Manusia

🔴 1. SyntaxError (Salah Ejaan/Tanda Baca)

🤖 Maksud Komputer:

"Bos, ejaannya salah nih! Coba cek lagi, ada tanda baca yang kurang gak?"

💊 Cara Mengobati:

Cek baris kode Anda. Biasanya Anda lupa menaruh tanda kutip ("), lupa tutup kurung ), atau lupa tanda titik dua (:).

🔴 2. NameError (Manggil Nama Ga Dikenal)

🤖 Maksud Komputer:

"Bos, Anda nyuruh saya nyari data bernama 'Budi', tapi saya cari-cari di sistem gak ada yang namanya 'Budi'!"

💊 Cara Mengobati:

Ini biasanya karena Anda Typo (salah ketik). Kemarin Anda menamai kotaknya nama_tamu, tapi sekarang Anda malah memanggil nama_tammu. Cek lagi ejaan nama variabelnya!

🔴 3. TypeError (Salah Wujud Data)

🤖 Maksud Komputer:

"Bos, Anda nyuruh saya mencampur Air dengan Minyak! Ya gak bisa bersatu dong!"

💊 Cara Mengobati:

Ini terjadi karena Anda mencoba mengawinkan dua jenis data yang menolak digabung. Contoh paling sering: Anda mencoba menjumlahkan Teks dengan Angka (Misal: "Budi" + 10). Gunakan alat sulap int() atau str() di Bab 2 untuk menyamakan wujud mereka dulu.

🔴 4. IndentationError (Spasi Berantakan)

🤖 Maksud Komputer:

"Bos, Python itu penderita OCD (Obsessive Compulsive Disorder). Kalau spasinya gak rata, saya gak mau kerja!"

💊 Cara Mengobati:

Di Bab 3 dan 4 (if, for, while), kita wajib memajukan teks menjorok ke dalam (di-Tab). Kalau ada satu baris saja yang kurang maju atau kelebihan spasi, Python akan ngambek. Rapikan lagi spasi Anda supaya sejajar!

🔴 5. IndexError (Salah Nomor Laci)

🤖 Maksud Komputer:

"Bos, Anda nyuruh saya ngambil barang di laci nomor 10, padahal lemarinya cuma punya 3 laci! Mau nyari di mana?"

💊 Cara Mengobati:

Ini terjadi saat Anda membongkar wadah data (seperti List di Bab 1). Ingat kembali bahwa komputer mulai berhitung dari angka 0. Kalau isi wadah Anda ada 3 barang, maka nomor lacinya hanya 0, 1, dan 2. Jangan pernah mencari laci nomor 3!

🔴 6. KeyError (Kunci Brankas Salah)

🤖 Maksud Komputer:

"Bos, Anda nyuruh buka laci pakai kunci berlabel 'Alamat', tapi saya ubek-ubek di sistem cuma ada label 'Nama' dan 'Umur'!"

💊 Cara Mengobati:

Sangat sering terjadi saat Anda menarik data AI dari internet (bentuk JSON/Dictionary). Komputer itu kaku; kalau label kuncinya "umur" (huruf kecil), Anda tidak bisa memanggilnya dengan "Umur" (huruf besar). Cek lagi daftar nama kunci yang tersedia di data Anda!

🔴 7. ValueError (Isi Tidak Sesuai Kodrat)

🤖 Maksud Komputer:

"Bos, Anda nyuruh saya ngubah kotak teks ini jadi angka mutlak. Boleh aja sih, tapi di dalam kotaknya tertulis kata 'Budi'! Tolong kasih tahu gimana caranya 'Budi' ditambah lima?!"

💊 Cara Mengobati:

Berbeda dengan TypeError (salah alat), ValueError terjadi karena alatnya benar tapi isinya tidak masuk akal. Pastikan jika Anda ingin memaksa teks menjadi angka pakai int(), teks tersebut memang benar-benar berisi angka (contoh: "100"), bukan huruf atau abjad.

🔴 8. AttributeError (Maksa Fitur yang Gak Ada)

🤖 Maksud Komputer:

"Bos, Anda nyuruh sepeda motor untuk terbang! Ya gak bisa dong, dia kan diciptakan gak punya sayap!"

💊 Cara Mengobati:

Terjadi saat Anda menyuruh sebuah benda melakukan Aksi (Bab 2) yang tidak dia miliki. Misalnya, Anda mencoba menambahkan daftar nama baru pakai aksi .append(), tapi ternyata benda yang Anda pegang adalah sebuah Teks murni, bukan sebuah List (Daftar). Pastikan Anda tahu benda apa yang sedang Anda pegang!

🔴 9. ModuleNotFoundError / ImportError (Alat Belum Dibeli)

🤖 Maksud Komputer:

"Bos, Anda nyuruh Koki memotong daging pakai gergaji mesin (import gergaji), padahal bos belum pernah beli gergajinya ke toko bangunan!"

💊 Cara Mengobati:

Anda mencoba meminjam alat (library) orang lain menggunakan import, tapi alat itu belum ada di laptop Anda.

Luka ini TIDAK BISA diobati pakai Kotak P3K (try-except)! Anda harus mematikan program, buka Terminal, lalu ketik pip install nama_alat_nya untuk mendownloadnya terlebih dahulu.

🔴 10. FileNotFoundError (Buku Resepnya Hilang)

🤖 Maksud Komputer:

"Bos, Anda nyuruh saya baca buku panduan perusahaan yang namanya data_karyawan.pdf, tapi saya cari di laci, di meja, di gudang... filenya gak ada tuh!"

💊 Cara Mengobati:

Ini biasanya karena 2 hal: (1) Anda salah ketik nama filenya (misal: Data_karyawan.pdf pakai huruf kapital), atau (2) File tersebut Anda taruh di folder yang berbeda dengan tempat program Python Anda berjalan. Pastikan letak filenya berdekatan atau tuliskan alamat lengkap foldernya (contoh: C:/Dokumen/data_karyawan.pdf).

🔴 11. TimeoutError (Koki Pingsan Nungguin Bahan)

🤖 Maksud Komputer:

"Bos, saya udah nelepon supplier daging (Server API) buat minta bahan. Tapi udah nungguin 2 menit gak diangkat-angkat. Daripada saya diem aja kaya patung, mending saya matikan sistemnya sekarang!"

💊 Cara Mengobati:

Ini BUKAN salah kode Anda! Ini terjadi karena koneksi internet Anda sedang jelek, atau server penyedia AI (seperti OpenAI/Gemini) sedang down/kepenuhan orang.

Sangat disarankan membungkus panggilan API Anda dengan P3K (try-except) agar saat internet putus, aplikasi Anda tidak langsung hancur, melainkan memunculkan pesan ramah: "Maaf, server AI sedang sibuk. Silakan coba lagi nanti."

🔴 12. JSONDecodeError (Bahasa Alien)

🤖 Maksud Komputer:

"Bos, saya disuruh nerjemahin paket data dari internet (API) pakai kamus JSON. Tapi pas saya buka, isinya berantakan bukan bahasa JSON yang saya ngerti!"

💊 Cara Mengobati:

Saat aplikasi Anda "ngobrol" dengan server luar (seperti ChatGPT), mereka sepakat memakai bahasa rahasia bernama JSON (bentuknya mirip Dictionary di Bab 1). Error ini muncul kalau server membalas dengan teks biasa atau format yang salah. Biasanya ini pertanda bahwa Anda salah memasukkan alamat API, salah password (API Key), atau server tersebut memang sedang rusak. Periksa kembali sambungan API Anda!

🛡️ Sudah Tahu Jenis-jenis Error?

Jangan biarkan program Anda langsung mati (crash) saat error terjadi. Pelajari cara menjinakkan error secara elegan menggunakan jurus Try-Except.
